## How it works

## Adding a recipe

1. Create a new `.yaml` file in the appropriate `data/<cuisine>/` subfolder.
   Name the file after the recipe id (e.g. `data/korean/kimchi-fried-rice.yaml`).

2. Follow the schema in `SCHEMA.yaml`. Required fields: `id`, `name`, `cuisine`, `difficulty`, `time`.

3. Run `./deploy.sh` (or `./deploy.sh "Add kimchi fried rice"`) to build and publish.

That's it вЂ” no other steps needed.

---

## Deploying

```bash
./deploy.sh                          # auto commit message
./deploy.sh "Add banana bread"       # custom commit message
```

The script:
- Runs `build.py` to regenerate `recipes.json`
- Stages all changes (`git add -A`)
- Commits (skips if nothing changed)
- Pushes to GitHub в†’ GitHub Pages picks it up in ~30 seconds

First-time setup only: make the script executable with `chmod +x deploy.sh`.

## Recipe schema quick reference

name: Display Name

description: One or two sentences.
